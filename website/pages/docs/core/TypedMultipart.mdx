Request body decorator of `multipart/form-data`.

`@TypedMultipart.Body()` is a request body decorator function for the `multipart/form-data` content type. It automatically casts property type following its DTO definition, and performs the type validation too.

As you can see from the below code, `@TypedMultipart.Body()` function is much easier and type safer than `@UploadFile()` of NestJS. Also, if you're considering the [SDK library](../sdk/sdk) generation, only `@TypedMultipart.Body()` is supported. Therefore, I recommend you to utilize `@TypedMultipart.Body()` instead of the `@UploadFile()` function.

Of course, as every features of `nestia` does, you don't need to define extra schema definitions for the [Swagger Documents](../sdk/swagger) generation. `@TypedMultipart.Body()` will do everything just by analyzing your TypeScript types.

import { TypedMultipart, TypedRoute } from "@nestia/core";
import { Controller } from "@nestjs/common";
import { tags } from "typia";

@Controller("multipart")
export class MultipartController {
  @TypedRoute.Post()
  public post(
    @TypedMultipart.Body() body: IMultipart
  ): void {
    body;
  }
}

export interface IMultipart {
  id: string & tags.Format<"uuid">;
  strings: string[];
  number: number;
  integers: Array<number & tags.Type<"int32">>;
  file: File;
  files: File[];
}
